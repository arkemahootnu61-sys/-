# 全栈

### html，css，js三件套分别都干了什了

- html是骨骼，css是皮肤，js是皮肤
- css有三种表示方法，按照标签，class或者id，id就相当于是身份证，class相当于是职业
- js是放在scripts里面的，当看到这个scripts就意味着应该执行代码了
- 浏览器渲染的时候遇到scc或者js都会进行下载再进行下一步，所以一般都会把css与js单独写成文件，然后在head里面引入，这样会减少网站出来了但是点击没有反映或者是网页一片空白没有反映的情况
- DOM树，浏览器解析玩html后，会形成一个DOM树，每一个标签都会成为树上的一个节点，嵌套关系就是树上的父子关系
- F12是开发者工具，上面的更改都是临时更改，刷新就会重新渲染，恢复原样

# 加密技术

- 对称加密，就是加密和解密都是用同一套密钥
- 非对称加密就是加密和解密不适用一套密钥，接受者通过哈希算法先生成一串私钥，再通过私钥生成公钥，通过公钥进行加密，通过私钥进行解密，除此之外，还可以通过私钥进行数字签名，通过公钥来进行鉴定

#### CA证书

- 这里拿银行举例子，银行给你发送消息让你填写账号密码，那你是如何证明这是银行发送的消息而不是·中间有黑客拦截了
- 答案就是CA机构会颁发一些证书，私钥是保存在机构的手里，那么问题来了
  - 服务器是怎么 拿到CA证书呢，那就是CA机构会验证你是否拥有该域名，如果是你的，就会给你颁发
  - 那客户端是如何拿到公钥呢，答案是，他会把一些主流的CA公要提前安装到你的浏览器
- 证书信任链
  - 就是分为根CA证书以及中间CA证书，构建信任链
